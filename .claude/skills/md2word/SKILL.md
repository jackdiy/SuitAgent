# Markdown转Word文档转换工具

## 概述

此工具将Markdown文档转换为符合中国法律文书标准的专业格式Word文档。当您需要将Markdown格式的法律文档、合同、协议等转换为正式的Word格式时，此工具会自动应用专业的排版标准。

## 功能特性

### 格式标准
- **字体规范**: 中文使用仿宋_GB2312，英文使用Times New Roman
- **字号标准**: 正文小四(12pt)，一级标题小三(15pt)
- **段落格式**: 1.5倍行距，首行缩进2字符，两端对齐
- **页面设置**: A4纸张，上下边距2.54cm，左右边距3.18cm
- **标题格式**: 一级标题居中加粗，二级标题加粗左对齐

### Markdown元素支持
- ✅ 标题转换（# ## ###）
- ✅ 段落和列表
- ✅ 表格
- ✅ 引用块
- ✅ 代码块
- ✅ **加粗**、*斜体*、~~删除线~~ 等文本格式
- ✅ 自动页码（1/x格式）
- ✅ 甲乙双方签字区域

## 什么时候使用

当您表达以下需求时，此工具会自动被调用：

- "请将这个Markdown文件转换为Word文档"
- "把保密协议.md转成Word格式"
- "需要把技术文档输出为.docx文件"
- "将合同文档转换为专业格式的Word文档"

## 转换示例

### 输入：Markdown文档

```markdown
# 保密协议

甲方：__________________________
乙方：__________________________

鉴于乙方将为甲方提供外部加工服务，可能接触到甲方的商业秘密，双方本着平等互利的原则，就保密事宜达成如下协议：

## 第一条 定义

1. "商业秘密"是指不为公众所知悉，能为甲方带来经济利益，具有实用性并经甲方采取保密措施的技术信息和经营信息。

2. "技术资料"属于甲方的技术秘密，是商业秘密的组成部分。

## 第二条 保密义务

乙方对甲方提供的全部商业秘密负有严格的保密义务。
```

### 输出：Word文档

转换后的文档将自动应用：

- **页面设置**: A4纸张，标准边距
- **字体**: 全文使用仿宋_GB2312
- **标题格式**:
  - 一级标题：居中，小三号，加粗
  - 二级标题：左对齐，小四号，加粗，首行缩进
- **正文格式**: 小四号，1.5倍行距，首行缩进2字符
- **列表格式**: 自动编号，首行缩进
- **签字区域**: 自动添加甲乙双方签字区域
- **页码**: 页脚显示（1/x格式）

- 仿宋_GB2312字体
- 小四字号
- 1.5倍行距
- 首行缩进2字符

1. 有序列表项
2. 第二项
3. 第三项

## 技术实现

### 转换流程
1. 自动读取Markdown文件
2. 解析文档结构（标题、段落、列表等）
3. 识别Markdown格式元素
4. 转换为Word文档对象模型
5. 应用中国法律文书格式标准
6. 生成.docx文件

### 依赖要求
- Node.js >= 14.0.0
- npm包：docx（用于生成Word文档）

### 输出文件位置
生成的Word文档默认保存在与输入Markdown文件相同的目录下，文件名相同但扩展名为`.docx`。

## 格式输出特性

转换后的文档包含以下专业元素：

1. **标准法律文书格式**
   - 统一的仿宋_GB2312字体
   - 标准字号（小四/小三）
   - 规范的行距和缩进

2. **专业排版**
   - 1.5倍行距
   - 首行缩进2字符（24pt）
   - 两端对齐

3. **页面设置**
   - A4纸张 (21cm × 29.7cm)
   - 上下边距：2.54cm
   - 左右边距：3.18cm

4. **自动附加元素**
   - 页脚页码（当前页/总页数格式）
   - 甲乙双方签字区域
   - "【以下无正文】"标记

## 故障排除

### 问题1：找不到docx库
```bash
npm install docx
```

### 问题2：转换失败
可能的原因和解决方案：
- **文件不存在**: 检查输入文件路径是否正确
- **编码问题**: 确保Markdown文件为UTF-8编码
- **权限问题**: 确保脚本有读取和写入权限

### 问题3：格式不符合预期
- 检查输入Markdown语法是否正确
- 确认使用标准的Markdown格式标记
