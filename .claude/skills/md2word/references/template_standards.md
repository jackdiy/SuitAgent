# Word模板创建和使用标准

## 模板文件规范

### 文件命名
- 建议使用描述性名称，如：
  - `法律模板.docx`
  - `合同模板.docx`
  - `报告模板.docx`
  - `会议纪要模板.docx`

### 自动发现规则
程序会自动查找同目录下的.docx文件，优先选择包含以下关键词的文件：
- "模板"
- "template"
- "标准"
- "standard"

## 模板内容设置

### 必须设置的样式
1. **页面设置**:
   - 纸张大小: A4
   - 页边距: 上2.54cm，下2.54cm，左3.18cm，右3.18cm

2. **默认字体**:
   - 中文字体: 仿宋_GB2312
   - 西文字体: Times New Roman
   - 字号: 12pt
   - 颜色: 黑色

3. **段落样式**:
   - 正文: 首行缩进2字符，1.5倍行距
   - 标题1: 居中，加粗，小三号
   - 标题2: 左对齐，加粗，小四号

### 可选设置
- 页眉页脚内容
- 公司logo
- 水印
- 特殊的段落样式

## 模板使用流程

### 创建模板
1. 新建Word文档
2. 设置页面布局和页边距
3. 定义字体和段落样式
4. 添加通用的页眉页脚
5. 保存为模板文件

### 应用模板
```bash
# 基本使用（自动发现模板）
python md2word.py input.md output.docx

# 指定模板文件
python md2word.py input.md output.docx template.docx
```

### 模板处理机制
1. **保留设置**: 页面设置、默认字体、基础样式
2. **清空内容**: 所有文本内容和表格
3. **应用格式**: 按照Markdown内容重新生成文档

## 模板示例

### 法律文档模板
```
页面设置: A4，标准页边距
默认字体: 仿宋_GB2312 12pt
标题样式: 一级标题居中加粗，二级标题左对齐加粗
页眉: [公司名称] 法律文书
页脚: 第 [页码] 页 共 [总页数] 页
```

### 合同模板
```
页面设置: A4，标准页边距
默认字体: 仿宋_GB2312 12pt
特殊样式: 条款编号样式，签名区域样式
页眉: 合同编号: [编号]
页脚: [公司名称] 版权所有
```

### 技术报告模板
```
页面设置: A4，标准页边距
默认字体: 仿宋_GB2312 12pt
标题样式: 层级分明的标题样式
页眉: [项目名称] 技术报告
页脚: 生成日期: [日期]
```

## 模板兼容性

### 支持的Word版本
- Word 2007及以上版本
- 兼容.docx格式
- 不支持旧的.doc格式

### 样式继承
- 保留用户定义的样式
- 保留页面设置和页边距
- 保留字体和颜色设置
- 保留页眉页脚设置

## 最佳实践

### 模板设计原则
1. **简洁性**: 保持模板简洁，避免过多装饰
2. **一致性**: 确保格式设置的一致性
3. **通用性**: 适用于多种文档类型
4. **专业性**: 符合正式文档的标准

### 模板测试
1. 创建测试Markdown文件
2. 使用模板进行转换
3. 检查格式应用情况
4. 调整模板设置
5. 重复测试直到满意

### 维护建议
1. 定期更新模板文件
2. 备份重要模板
3. 记录模板版本
4. 收集用户反馈
5. 持续优化格式设置
