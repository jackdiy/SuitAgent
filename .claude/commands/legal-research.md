# legal-research 命令配置 v1.2

## 版本记录
- **v1.2** (2025-11-12) → **增强链接导航和内嵌链接**
- **v1.1** (2025-11-12) → 精简版本：保留核心功能
- **v1.0** (2025-11-12) → 初始版本

## 命令功能概述
自动分析法律材料，生成结构化问题研究清单，为每个问题创建独立详细文档，提供内嵌式参考链接导航。

## 核心改进 (v1.2)
### 🎯 智能链接系统
- **清单文档内置导航：** 问题研究清单中每个问题都有直接链接跳转到详细文档
- **内嵌式参考链接：** 参考链接直接嵌入到相关内容的旁边，而不是集中在文档末尾
- **上下文相关链接：** 每个链接都标注其用途（如"查看法条原文"、"参考判例"）
- **三级链接体系：** 问题清单 → 详细文档 → 上下文内嵌资源

## 输入格式

### 使用方式
```
/legal-research @文档路径.md
研究要求：{具体要求}

或者

/legal-research
[粘贴材料内容]
研究要求：{具体要求}
```

### 支持材料
- 聊天记录、会谈纪要
- 案例描述、法律条文
- 网页链接、学术文章

## 处理流程
1. **分析材料** - 提取法律要素和争议焦点
2. **识别问题** - 分类：核心⭐⭐⭐⭐⭐、重要⭐⭐⭐⭐、延伸⭐⭐⭐
3. **生成清单** - 创建结构化总览文档（含直接链接）
4. **生成文档** - 为每个问题创建独立详细文档（含内嵌链接）
5. **整合链接** - 法律条文、案例、机构等参考资源内嵌到上下文

### 文档内链接布局规则

**1. 链接位置：**
- **法条引用旁边：** `⚖️ [35 U.S.C. § 171 - 设计专利法定依据](https://链接)`
- **案例引用旁边：** `📚 [Egyptian Goddess v. SWISA - 判例全称](https://链接)`
- **工具引用旁边：** `🛠️ [Brand Registry门户](https://链接)`
- **政策引用旁边：** `🔗 [亚马逊知识产权政策](https://链接)`
- **计算工具旁边：** `🧮 [损害赔偿计算器](https://链接)`

**2. 链接标注约定：**
- 📄 → 指向本地详细文档
- 🔗 → 指向外部网页资源
- ⚖️ → 指向法律条文
- 📚 → 指向判例或案例
- 🛠️ → 指向工具或服务
- 🧮 → 指向计算工具
- ⚠️ → 指向风险提示

**3. 避免重复链接原则：**
- 清单中已有 `[📄 问题名](./文件.md)` 链接
- 详细文档底部导航中只需写 `[📄 下一问题](./文件.md)` **无需再加文字"下一问题：xxx"**
- 直接使用链接标题即可，格式简洁

## 质量标准

### 问题研究标准
- **全面性：** 识别所有有价值法律问题
- **准确性：** 符合法律术语规范
- **实用性：** 具有实际指导意义
- **系统性：** 问题间有逻辑关联

### 文档要求
- 每个问题独立详细文档
- 包含操作指南和步骤
- 提供内嵌式参考链接（上下文相关）
- 标注重要程度
- 包含成本效益分析
- 提供风险提示

### 链接质量标准
- **可用性：** 所有链接必须真实可访问
- **相关性：** 链接内容与上下文直接相关
- **权威性：** 优先使用官方权威链接
- **多样性：** 包括法条、案例、机构、工具等类型
- **标注清晰：** 使用emoji明确标识链接类型
- **避免重复：** 清单中已有的链接，详细文档中不再重复描述

### 导航质量标准
- **清单导航：** 每个问题都有直接 `[📄 详细文档](./文件.md)` 链接
- **快速导航：** 提供按类型和按阶段两种导航方式
- **底部导航：** 简洁的导航链接，无冗余文字
- **上下文导航：** 关联问题之间的导航清晰明了

## 注意事项

### 输入要求
- 优先使用MD格式
- 包含关键事实和法律要素
- 明确研究重点和要求

### 链接管理优化 (v1.2)
- **避免链接堆砌：** 每个段落最多2-3个相关链接
- **链接上下文化：** 链接前有明确说明，链接后有简要解释
- **层级分明：** 本地文档链接 vs 外部资源链接区分清楚
- **导航友好：** 提供返回和跳转选项
- **避免重复：** 清单中链接已在，底部导航简洁呈现

### 链接验证检查清单
在生成文档后，检查：
- [ ] 所有本地链接路径正确且文件存在
- [ ] 所有外部链接可正常访问
- [ ] 链接与上下文内容匹配
- [ ] 链接标注（emoji）清晰易懂
- [ ] 清单中每个问题都有直接链接
- [ ] 详细文档底部导航简洁无重复

### 文档生成工作流
1. **生成清单文档**
   - 添加快速导航区
   - 每个问题添加 `[📄 详细文档](./文件.md)` 链接
   - 在列表中直接显示关键数据（成功率、成本等）
   - 添加底部导航区

2. **生成详细文档**
   - 在相关内容旁边内嵌参考链接
   - 使用emoji标注链接类型
   - 移除末尾集中参考资源区
   - 底部简洁导航（无冗余文字）

3. **最终检查**
   - 验证所有链接可用性
   - 确认导航链接功能
   - 检查避免重复链接
